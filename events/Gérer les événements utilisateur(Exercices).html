Exercice 1 : La carte produit interactive MouseEvents

Contexte professionnel : Vous développez le front-end d'un site e-commerce moderne. Pour épurer l'interface, le bouton "Ajouter au panier" n'apparaît que lorsque l'utilisateur survole l'image du produit avec sa souris.
Code fourni (HTML/CSS) :
<div class="product-card" id="card">
    <img src="sneaker.jpg" alt="Sneaker X" class="product-image">
    <div class="product-info">
        <h3>Sneaker X-Pro</h3>
        <p>1200 DH</p>
    </div>
    <button id="addBtn" class="add-to-cart-btn hidden">Ajouter au panier</button>
</div>

<style>
    .product-card { width: 250px; border: 1px solid #eee; border-radius: 12px; padding: 15px; position: relative; overflow: hidden; transition: box-shadow 0.3s; }
    .product-image { width: 100%; border-radius: 8px; }
    .hidden { opacity: 0; transform: translateY(20px); pointer-events: none; transition: all 0.3s ease; }
    .visible { opacity: 1; transform: translateY(0); pointer-events: auto; }
    .add-to-cart-btn { width: 100%; padding: 10px; background: #000; color: #fff; border: none; border-radius: 6px; cursor: pointer; margin-top: 10px; }
</style>
Travail demandé (JavaScript) :
Écrivez le script permettant de :
  • Détecter l'entrée de la souris (mouseenter) sur l'élément .product-card pour enlever la classe hidden et ajouter la classe visible au bouton.
  • Détecter la sortie de la souris (mouseleave) pour faire l'inverse (cacher le bouton).
  • Ajouter un événement de clic sur le bouton qui affiche une alerte : "Produit ajouté au panier !".

Exercice 2 : Système de notation par étoiles MouseEvents

Contexte professionnel : Pour un site de réservation de restaurants, vous devez créer un composant permettant aux utilisateurs de laisser une note de 1 à 5 étoiles. Le composant doit réagir au survol pour montrer la note potentielle avant de cliquer.
Code fourni (HTML/CSS) :
<div class="rating-container" id="starRating">
    <span class="star" data-value="1">★</span>
    <span class="star" data-value="2">★</span>
    <span class="star" data-value="3">★</span>
    <span class="star" data-value="4">★</span>
    <span class="star" data-value="5">★</span>
</div>
<p id="ratingText">Note : 0/5</p>

<style>
    .star { font-size: 30px; color: #ccc; cursor: pointer; transition: color 0.2s; }
    .star.hovered { color: #ffd700; } /* Jaune */
    .star.selected { color: #ffaa00; } /* Orange foncé */
</style>
Travail demandé (JavaScript) :
Écrivez le script permettant de :
  • Au survol (mouseover) d'une étoile, ajouter la classe hovered à cette étoile et à toutes celles qui la précèdent.
  • À la sortie de la souris (mouseout) du conteneur #starRating, retirer la classe hovered de toutes les étoiles.
  • Au clic (click) sur une étoile, mémoriser la note, appliquer la classe selected aux bonnes étoiles, et mettre à jour le texte #ratingText (ex: "Note : 3/5").

Exercice 3 : La "Command Palette" type GitHub KeyboardEvents

Contexte professionnel : Les applications SaaS modernes (comme Discord ou GitHub) permettent aux "power users" d'utiliser des raccourcis clavier. Vous devez ouvrir une barre de recherche rapide lorsque l'utilisateur appuie sur la touche Ctrl et la touche K en même temps.
Code fourni (HTML/CSS) :
<div id="searchModal" class="modal hide">
    <div class="modal-content">
        <input type="text" id="searchInput" placeholder="Rechercher... (Échap pour fermer)">
    </div>
</div>
<p>Appuyez sur <strong>Ctrl + K</strong> pour ouvrir la recherche.</p>

<style>
    .modal { position: fixed; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0,0,0,0.5); display: flex; justify-content: center; align-items: flex-start; padding-top: 10vh; }
    .modal-content { background: white; padding: 20px; border-radius: 8px; width: 400px; }
    .modal input { width: 100%; padding: 10px; font-size: 16px; border: 1px solid #ccc; border-radius: 4px; }
    .hide { display: none; }
</style>
Travail demandé (JavaScript) :
Écrivez le script permettant de :
  • Écouter l'événement keydown sur l'ensemble du document.
  • Si les touches Ctrl et K sont pressées simultanément, empêcher le comportement par défaut du navigateur (preventDefault() pour éviter que la barre d'URL prenne le focus), enlever la classe hide de la modale et donner le focus (focus()) au champ #searchInput.
  • Si la touche Escape est pressée, recacher la modale en ajoutant la classe hide.

Exercice 4 : Saisie de code de vérification (OTP) KeyboardEvents

Contexte professionnel : Lors de l'authentification à double facteur (2FA), l'utilisateur doit saisir un code à 4 chiffres reçu par SMS. Pour une meilleure expérience (UX), le curseur doit passer automatiquement à la case suivante après la frappe d'un chiffre.
Code fourni (HTML/CSS) :
<div class="otp-container" id="otpForm">
    <input type="text" class="otp-input" maxlength="1" pattern="[0-9]">
    <input type="text" class="otp-input" maxlength="1" pattern="[0-9]" disabled>
    <input type="text" class="otp-input" maxlength="1" pattern="[0-9]" disabled>
    <input type="text" class="otp-input" maxlength="1" pattern="[0-9]" disabled>
</div>

<style>
    .otp-container { display: flex; gap: 10px; }
    .otp-input { width: 40px; height: 50px; font-size: 24px; text-align: center; border: 2px solid #ccc; border-radius: 8px; outline: none; transition: border-color 0.3s; }
    .otp-input:focus { border-color: #007bff; }
</style>
Travail demandé (JavaScript) :
Écrivez le script permettant de :
  • Cibler tous les champs .otp-input.
  • Sur l'événement keyup, si l'utilisateur saisit un chiffre, activer le champ suivant (enlever l'attribut disabled) et lui donner le focus.
  • Si l'utilisateur appuie sur la touche Backspace (Retour arrière), effacer le contenu du champ actuel, désactiver ce champ, et redonner le focus au champ précédent.

Exercice 5 : Validation d'un formulaire d'inscription Form Events

Contexte professionnel : Vous devez valider les données d'un formulaire côté client avant de les envoyer au serveur. Cela évite des requêtes inutiles et donne un retour immédiat à l'utilisateur si les règles de sécurité ne sont pas respectées.
Code fourni (HTML/CSS) :
<form id="registerForm" class="modern-form">
    <div class="form-group">
        <label>Nom d'utilisateur</label>
        <input type="text" id="username">
        <span class="error-msg" id="errorName"></span>
    </div>
    <div class="form-group">
        <label>Mot de passe</label>
        <input type="password" id="password">
        <span class="error-msg" id="errorPwd"></span>
    </div>
    <button type="submit">Créer mon compte</button>
</form>
<div id="successMsg" class="success hide">Inscription réussie !</div>

<style>
    .modern-form { max-width: 300px; display: flex; flex-direction: column; gap: 15px; }
    .form-group { display: flex; flex-direction: column; }
    input { padding: 8px; border: 1px solid #ccc; border-radius: 4px; }
    input.invalid { border-color: red; }
    .error-msg { color: red; font-size: 12px; height: 15px; margin-top: 5px; }
    .success { color: green; font-weight: bold; margin-top: 20px; }
    .hide { display: none; }
    button { padding: 10px; background: #28a745; color: white; border: none; border-radius: 4px; cursor: pointer; }
</style>
Travail demandé (JavaScript) :
Écrivez le script permettant de :
  • Écouter l'événement submit sur le formulaire.
  • Interrompre la soumission par défaut de la page (preventDefault()).
  • Valider les champs selon ces règles :
    • Le Nom d'utilisateur doit contenir au moins 5 caractères. Si ce n'est pas le cas, afficher "Le nom doit contenir au moins 5 caractères" dans #errorName et ajouter la classe invalid au champ.
    • Le Mot de passe doit contenir au moins 8 caractères. Sinon, afficher "Mot de passe trop court" dans #errorPwd et ajouter la classe invalid.
  • Si toutes les validations sont respectées, cacher le formulaire et afficher la div #successMsg. (Pensez à réinitialiser les erreurs à chaque nouvelle tentative de soumission).
